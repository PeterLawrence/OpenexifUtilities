    Introduction

    OpenExif is an object-oriented interface to Exif formated JPEG image files. The toolkit allows creating, reading, and modifying the metadata in the Exif file. It also provides means of getting and setting the main image and the thumbnail image. OpenExif is also extensible, and Application segments can be added.

    Toolkit features

    • Source code available under Open Source license enables users to contribute to the toolkit
    • Open source license allows for toolkit usage in commercial applications
    • Provides an extensible framework for integrating other compression schemes.
    • Provides an extensible framework for Application Segments allowing users to implement their own private application segments and register them in the toolkit.

    Platforms

    • Win32 using VC6, VC7.0 or VC7.1
    • Unix/Linux/MacOSX using Gnu gcc

    Build Note

    Building on Windows with VC6 or VC7 (.Net)

    Building openexif requires:

    • NT 3.51 or higher, 98, 2000, XP.
    • VC6, VC7.0, VC7.1

    Download the IJG Jpeg toolkit from the IJG website or the OpenExif website. You will need to copy (or rename) jconfig.vc to jconfig.h. You may also need to change the definition of INT32 in jmorecfg.h from long to signed int to get rid of a redefinition error when building OpenExif. Then, build it with VC's nMake (prompt> nMake makefile.vc). This will create libjpeg.lib. OpenExif is expecting jpeg.lib, so make a copy of libjpeg.lib and rename it to jpeg.lib (or change OpenExif's project settings later).

    Set environment variable JPEGROOT to point to the Jpeg toolkit (for example, C:\Toolkits\jpeg-6b).

    To build openexif from its source , you will need to open the workspaces containing the projects. For VC6, they are:

    • projects\Win32\Vc6.0\BuildAllStatic.dsw: for building static libraries and its test program.
    • projects\Win32\Vc6.0\BuildAllDynamic.dsw: for building dlls and its test program.

    For VC7.0 or VC7.1, they are:

    • projects\Win32\VC7.0\BuildAllStatic.sln: for building static libraries and its test program.
    • projects\Win32\VC7.0\BuildAllDynamic.sln: for building dlls and its test program.

    If you want to include the openexif project into your application's workspace, you need to pick up the project files. For VC6:

    • projects\Win32\Vc6.0\OpenExif.dsp: for building static libraries.
    • projects\Win32\Vc6.0\OpenExifi.dsp: for building dlls.

    For VC7.0 or VC7.1:

    • projects\Win32\VC7.0\OpenExif.vcproj: for building static libraries.
    • projects\Win32\VC7.0\OpenExifi.vcproj: for building dlls.

    Building on Unix/Linux/MacOSX

    For building openexif the following compilers and toolks have been tested and are supported:

    • GNU g++ 2.95.3 and greater
    • autoconf 2.53, automake 1.6.3, libtool 1.4.2 and m4 1.4.1

    The steps are:

    • Download the IJG Jpeg toolkit from the IJG website or the OpenExif website. Then configure and build it: (prompt$ ./configure --prefix='somepathtoinstall', prompt$ make, prompt$ make install). NOTE: Some systems already have the IJG Jpeg toolkit installed. Search for jpeglib.h in /usr/include, for example.
    • Set environment variable JPEGROOT to point to your Jpeg toolkit. For example, if your system already has jpeglib.h in /usr/include and libjpeg.a in /usr/lib, then set JPEGROOT to /usr. Otherwise, set JPEGROOT to point to the directory where you installed the IJG Jpeg toolkit.
    • cd to the root directory of openexiff.
    • type ./configure and then ./make
